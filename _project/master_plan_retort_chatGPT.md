# Master Plan of Action: High-Fidelity Synthesis v2.1 (HFS — Hardened)

---

# **1. Executive Summary**

This plan defines a hardened evolution of the "Compounding Illusion" research project. It replaces linear summarization with a **Recursive Knowledge Architecture (RKA)** that is:

* **Temporally aware** (tracks evolution of ideas)
* **Conflict-explicit** (formalizes disagreements)
* **Empirically validated** (simulation-backed)
* **Structurally normalized** (ontology-driven)

The goal is to produce **Final Tome v4.0**, a system that is:

* Evidence-backed
* Contradiction-resolved
* Simulation-verified
* Modular and queryable

---

# **2. Core Architectural Upgrade**

## **2.1 The Problem: "The Flattening Effect" (v1.0 Failure Modes)**

The previous methodology failed due to:

* **Temporal Collapse:** No differentiation between legacy and modern claims
* **Unverified Math:** Extracted formulas without simulation
* **Implicit Contradictions:** No formal conflict detection
* **Ontology Drift:** Mixing incompatible concept types
* **Static Output Bias:** Knowledge frozen into non-evolving form

---

## **2.2 The Solution: Recursive Knowledge Architecture (RKA)**

RKA introduces:

1. **Claim-Centric Modeling** (all knowledge normalized into structured claims)
2. **Explicit Conflict Typing** (no implicit disagreements)
3. **Multi-Layer Validation** (math, system, player simulation)
4. **Ontology Enforcement** (strict entity structure)
5. **Separation of Concerns** (extraction ≠ arbitration ≠ design)

---

# **3. Core Data Model (CRITICAL ADDITION)**

## **3.1 Claim Schema (Mandatory Extraction Format)**

Every extracted insight MUST be represented as:

```
Claim:
  id:
  statement:
  domain: (economy / pacing / UX / monetization / system)
  level: (micro / system / meta)
  assumptions:
  variables:
  evidence_type: (empirical / theoretical / anecdotal)
  source:
  date:
  confidence: (high / medium / low)
```

---

## **3.2 Conflict Typology (Mandatory Classification)**

All contradictions MUST be classified:

```
Conflict Types:
  - Direct Contradiction
  - Scope Mismatch
  - Temporal Supersession
  - Optimization Divergence
```

---

## **3.3 Ontology Layer (Prevents Structural Drift)**

All extracted elements MUST map to:

```
Entities:
  - Resource
  - Generator
  - Modifier
  - Time Mechanic
  - Player Action
  - Monetization Hook
  - Feedback Loop
```

No extraction may bypass this structure.

# **4. Phase 1: Universal Corpus Re-Atomization (Revised)**

**Scope:** 20+ documents (Legacy + 2024–2026 Harvest)

---

## **4.1 Triple-Pass Extraction (Hardened)**

### **Pass 1 — Atomic Extraction**

Extract:

* Definitions
* Mechanics
* Systems
* Examples

All outputs MUST conform to:

* Claim Schema
* Ontology Mapping

---

### **Pass 2 — Delta Mapping (Formalized)**

#### For Legacy Documents:

* Establish baseline claims

#### For Modern Documents:

* Compare against baseline using:

```
Delta Classification:
  - Confirms
  - Extends
  - Contradicts
  - Replaces
```

All differences MUST reference:

* Claim IDs
* Conflict Type

---

### **Pass 3 — Precision Extraction (Normalized Math Layer)**

All equations MUST be:

* Transcribed into `math_ledger.md`
* Normalized:

  * consistent variable naming
  * explicit assumptions
  * unit-aware

If incomplete:

* mark as `LOW CONFIDENCE`

---

## **4.2 Extraction Confidence Handling**

Every extraction MUST include:

```
confidence:
  - high (explicit + consistent)
  - medium (partial or inferred)
  - low (ambiguous or incomplete)
```

The original single matrix is replaced with **three specialized systems**:

* Claims ↔ Sources

---

## **5.2 System Interaction Matrix**

* Mechanics ↔ Mechanics

## **5.3 Conflict Matrix**

* Claim ↔ Claim

* Explicit contradiction handling
